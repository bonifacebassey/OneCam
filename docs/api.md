# API Reference

Base URL: `http://localhost:8000` (or your server's LAN IP)

Interactive docs are auto-generated by FastAPI and available at `/docs` when the server is running.

---

## Cameras

### List cameras

```
GET /api/cameras
```

Returns all enabled cameras registered in `cameras.json`.

**Response 200**
```json
[
  {
    "id": "front-door",
    "label": "Front Door",
    "stream_url": "/api/stream/front-door",
    "snapshot_url": "/api/snapshot/front-door"
  }
]
```

---

### Live stream

```
GET /api/stream/{cam_id}
```

Streams a continuous MJPEG feed from the camera. Designed to be used as the `src` attribute of an `<img>` tag.

```html
<img src="/api/stream/front-door" />
```

**Response** — `multipart/x-mixed-replace; boundary=frame`

The generator closes cleanly when the client disconnects. If the camera is unreachable the generator ends immediately, causing the `<img>` element to fire `onerror` so the dashboard can display the offline overlay.

**Errors**

| Code | Meaning |
|---|---|
| 404 | Camera ID not found |

---

### Snapshot

```
GET /api/snapshot/{cam_id}?save=false
```

Fetches a single JPEG frame from the camera.

**Query parameters**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `save` | boolean | `false` | If `true`, saves the image to `snapshots/{cam_id}_{timestamp}.jpg` before returning it |

**Response 200** — `image/jpeg`

**Errors**

| Code | Meaning |
|---|---|
| 404 | Camera ID not found |
| 503 | Camera unreachable |

---

### Control camera

```
POST /api/cameras/{cam_id}/control
Content-Type: application/json
```

Forwards a parameter change to the camera's `/control` endpoint.

**Request body**
```json
{
  "variable": "framesize",
  "value": 8
}
```

**Supported variables**

| Variable | Type | Range | Description |
|---|---|---|---|
| `framesize` | int | 0–13 | Frame size (see firmware docs for values) |
| `quality` | int | 4–63 | JPEG quality — lower = better quality |
| `brightness` | int | -2 to 2 | Brightness adjustment |
| `contrast` | int | -2 to 2 | Contrast adjustment |
| `saturation` | int | -2 to 2 | Saturation adjustment |
| `hmirror` | int | 0 or 1 | Horizontal mirror |
| `vflip` | int | 0 or 1 | Vertical flip |
| `awb` | int | 0 or 1 | Auto white balance |
| `agc` | int | 0 or 1 | Auto gain control |
| `aec` | int | 0 or 1 | Auto exposure control |

**Response 200**
```json
{
  "status": "ok",
  "camera": "front-door",
  "variable": "framesize",
  "value": 8
}
```

**Errors**

| Code | Meaning |
|---|---|
| 404 | Camera ID not found |
| 422 | Missing or invalid request body |
| 503 | Camera unreachable |

## Snapshots

### List saved snapshots

```
GET /api/snapshots
```

Returns all JPEG files in the `snapshots/` directory, sorted newest first.

**Response 200**
```json
[
  {
    "filename": "front-door_20260509_143022.jpg",
    "url": "/api/snapshots/front-door_20260509_143022.jpg"
  },
  ...
]
```

---

### Get a saved snapshot

```
GET /api/snapshots/{filename}
```

Serves a saved snapshot file.

**Response 200** — `image/jpeg`

**Errors**

| Code | Meaning |
|---|---|
| 404 | File not found or not a `.jpg` |

## Events

### Motion alerts (SSE)

```
GET /api/events
Accept: text/event-stream
```

Server-Sent Events stream. The browser connects once and receives real-time motion alerts. Used internally by the dashboard via `EventSource`.

**Connection event** (on open)
```
: connected
```

**Motion event** (when motion is detected)
```
event: motion
data: {"camera_id": "front-door", "label": "Front Door", "timestamp": "20260509_143022", "snapshot_file": "front-door_20260509_143022.jpg"}
```

**Keepalive** (every 15 seconds, prevents proxy timeouts)
```
: keepalive
```

**Connecting from JavaScript**

```javascript
const es = new EventSource("/api/events");

es.addEventListener("motion", (e) => {
  const data = JSON.parse(e.data);
  console.log(`Motion on ${data.label} at ${data.timestamp}`);
  console.log(`Snapshot: /api/snapshots/${data.snapshot_file}`);
});
```

`EventSource` automatically reconnects on disconnect. The `X-Accel-Buffering: no` response header prevents nginx and Cloudflare Tunnel from buffering the stream.

## Health

```
GET /health
```

Liveness probe — useful for monitoring or confirming the server started correctly.

**Response 200**
```json
{
  "status": "ok",
  "cameras": 2
}
```

## Dashboard

```
GET /
```

Redirects to `/ui/` (the browser dashboard).

```
GET /ui/
```

Serves `static/index.html` — the browser dashboard.

## Error format

FastAPI returns errors in the standard format:

```json
{
  "detail": "Camera 'nonexistent' not found"
}
```
